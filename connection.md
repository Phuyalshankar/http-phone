# 🐬 Dolphin HTTP-Phone Connection Troubleshooting Guide (कनेक्सन गाइड)

यो गाइडले कोड सम्पादन वा नेटवर्क परिवर्तन गर्दा आउने कनेक्सन समस्याहरू (विशेष गरी "Offline" देखाउने समस्या) लाई सेकेन्डमै समाधान गर्न मद्दत गर्नेछ।

---

## 🔄 १. कार्यप्रणाली (How it Works)
हाम्रो एप्लिकेसन **Server-driven UI** ढाँचामा आधारित छ:
1. एन्ड्रोइड एपले कम्प्युटरको **Dev Server (TCP 7788)** सँग बाइनरी स्क्रिनहरू सिङ्क गर्छ।
2. ब्याकइन्ड **API & Realtime Gateway (Port 3000)** ले WebSocket (`/realtime`) र WebRTC सिग्नलिङ (`/phone`) ह्यान्डल गर्छ।
3. **Dynamic IP Detector**: कोडमा कतै पनि IP ठेगाना हार्डकोड गरिएको छैन। सर्भर र क्लाइन्टले कम्प्युटरको सक्रिय वाइफाइ IP स्वतः पत्ता लगाउँछन् र एन्ड्रोइड क्लाइन्टलाई पठाउँछन्।

---

## 🛠️ २. सेकेन्डमै कनेक्सन ठीक गर्ने चरणहरू (Quick Fix Steps)

यदि एप "Offline" देखियो वा जडान हुन सकेन भने, निम्न बुँदाहरू पालना गर्नुहोस्:

### चरण १: मोबाइल र कम्प्युटर एउटै वाइफाइ (WiFi) मा भएको सुनिश्चित गर्नुहोस्
- दुवै उपकरण एउटै राउटर वा हटस्पटमा जोडिएको हुनुपर्छ।

### चरण २: दुवै सर्भरहरू चलिरहेका छन् कि छैनन् जाँच गर्नुहोस्
PowerShell वा टर्मिनलमा निम्न कमान्डहरू चलिरहेका हुनुपर्छ:
- **ब्याकइन्ड (Port 3000)**: `npm run dev` (वा main runner)
- **Dev Server (Port 7788)**: यो ब्याकइन्डसँगै स्वतः सुरु हुन्छ।

यदि कुनै सर्भर रोकिएको छ भने, यसलाई पुन: सुरु गर्नुहोस्:
```bash
npm run dev
```

### चरण ३: सत्र फाइल (`.dolphin-session.json`) जाँच वा सफा गर्नुहोस्
कहिलेकाहीँ पुरानो सत्र (stale session) टोकनको कारणले लगइन विच्छेद हुन्छ।
- **फाइलको स्थान**: `d:\http-phone\frontend\http-phone\.dolphin-session.json`
- **सफा गर्ने तरिका**: यदि जडान हुँदै भएन भने, यो फाइललाई डिलिट गरिदिनुहोस् र मोबाइल एप पुन: सुरु गर्नुहोस्। यसले नयाँ लगइन टोकन सिर्जना गर्न मद्दत गर्छ।

### चरण ४: सक्रिय IP ठेगाना पत्ता लगाउनुहोस्
तपाईंको कम्प्युटरको अहिलेको IP के हो भनेर हेर्न टर्मिनलमा रन गर्नुहोस्:
```powershell
ipconfig
```
र वाइफाइ वा इथरनेट एडप्टरको `IPv4 Address` (जस्तै `192.168.1.10`) हेर्नुहोस्।

## ⚡ ३. द्रुत कमान्डहरू (Quick Commands Cheat Sheet)

यदि कोड सम्पादन गर्दा केही बिग्रियो वा कन्फ्युज हुनुभयो भने, यी कमान्डहरू चलाउनुहोस्:

| कार्य | कमान्ड | विवरण |
| :--- | :--- | :--- |
| **सर्भर पुन: सुरु गर्न** | `npm run dev` | ब्याकइन्ड र कम्पाइलर दुवै रिस्टार्ट गर्छ। |
| **एन्ड्रोइड बिल्ड गर्न** | `npm run build:android` | नयाँ APK फाइल तयार गर्छ। |
| **टाइपस्क्रिप्ट एरर जाँच** | `npx tsc --noEmit` | कोडमा टाइप एररहरू भए नभएको जाँच्न। |
| **ADB जडान जाँच** | `adb devices` | कम्प्युटरमा USB मार्फत मोबाइल जोडिएको छ कि छैन हेर्न। |

## ⚠️ ४. सम्भावित समस्या र समाधान (FAQs)

#### क. मोबाइल स्क्रिनमा "Offline" सूचक (WsIndicator) रातो छ:
- कम्प्युटरको ब्याकइन्ड सर्भर (Port 3000) बन्द हुन सक्छ। यसलाई `npm run dev` गरेर सुरु गर्नुहोस्।
- कम्प्युटरको फायरवाल (Firewall) ले Port 3000 वा Port 7788 लाईर् ब्लक गरेको हुन सक्छ। फायरवाल अस्थाई रूपमा बन्द गर्नुहोस् वा अनुमति दिनुहोस्।

#### ख. मोबाइलमा लगइन गर्दा "Session Expired" वा "Auth Error" आयो:
- सर्भरमा टोकन म्याद सकिएको हुन सक्छ। मोबाइल एपको डाटा क्लियर गर्नुहोस् वा `.dolphin-session.json` फाइल डिलिट गरेर पुन: लगइन गर्नुहोस्।

#### ग. नयाँ कोड परिवर्तनहरू मोबाइलमा देखिएनन् (Hot Reload भएन):
- Dev Server को पोर्ट ७७८८ जडान विच्छेद भएको हुन सक्छ। मोबाइल एपलाई पूर्ण रूपमा बन्द (Force Stop) गरेर पुनः खोल्नुहोस्।
